Personal financial data is transactional, not linguistic. Merchant strings such as "SQ *WHOLE FOODS 982" carry no semantic structure; dollar amounts lack categorical context; timestamps bear no relationship to budgets or goals. We present Artha (Sanskrit artha -- wealth, meaning, purpose), a framework that pre-computes a typed domain ontology of nine financial object classes and seven primary relationships before any LLM inference call. A 15-tool agentic layer built in Java 21 / Spring Boot 3.2 reasons over enriched typed objects rather than raw SQL rows. We release an open evaluation harness with 60 benchmark queries across 6 reasoning categories and 10 behavioral archetypes, scored by a held-out Claude Opus 4.7 instance as LLM-as-judge on a four-dimension rubric. Artha's full configuration achieves a 75.0% pass rate (3.67/5 mean score); a controlled four-condition ablation isolates a 26.7 percentage-point improvement over a raw-baseline configuration. A novel finding is that the typed tool layer is the dominant source of lift (-25.0 pp when bypassed), while enrichment data alone contributes less (-5.0 pp) because the tools gracefully recompute missing enrichments on demand. We report honest methodological compromises -- post-hoc detector threshold tuning on synthetic data, reference-date substitution, same-provider LLM judge, single-seed user per archetype, and deferred human inter-rater validation -- and release code, data schema, and evaluation harness at https://github.com/Tej-Katika/artha.
